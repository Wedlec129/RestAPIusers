
REST (Representational State Transfer) - это архитектурный стиль для разработки веб-сервисов, 
    который определяет набор ограничений и ориентированный на ресурсы подход к взаимодействию между 
    системами. RESTful API (или REST API) представляет собой интерфейс программирования приложений,
    который следует принципам REST.

Основные принципы REST API включают:

    1. Использование HTTP-методов, таких как GET, POST, PUT и DELETE, для выполнения различных операций.
    2. Использование URI (Uniform Resource Identifiers) для идентификации ресурсов в системе.
    3. Отсутствие состояния между запросами, что означает, что каждый запрос должен содержать всю 
        необходимую информацию для его обработки.
    4. Ориентированность на ресурсы, что означает, что данные в системе представлены в виде ресурсов, 
        доступ к которым осуществляется через URI.

RESTful API позволяет разработчикам создавать веб-сервисы, которые могут быть легко использованы и интегрированы с другими системами. Этот подход делает веб-сервисы более масштабируемыми, гибкими и легкими для понимания и использования.


CRUD - это аббревиатура, которая означает основные операции, выполняемые над данными в системе. Каждая буква в слове CRUD представляет одну из четырех основных операций:

    1. Create (Создание): Операция, позволяющая создавать новые данные в системе. 
        Это может включать создание новых записей в базе данных или добавление новых ресурсов в систему.

    2. Read (Чтение): Операция, позволяющая читать или извлекать данные из системы. 
        Это может включать чтение существующих записей из базы данных или получение 
        информации о существующих ресурсах в системе.

    3. Update (Обновление): Операция, позволяющая обновлять или изменять существующие 
        данные в системе. Это может включать изменение значений полей в базе 
        данных или обновление информации о существующих ресурсах.

    4. Delete (Удаление): Операция, позволяющая удалять существующие данные из системы. 
        Это может включать удаление записей из базы данных или удаление существующих 
        ресурсов из системы.

    CRUD представляет собой стандартный набор операций, которые используются для управления данными в различных типах систем, включая базы данных, веб-приложения, API и другие программные приложения. Эти операции являются основой для многих систем управления данными и обеспечивают основные возможности работы с данными.
